# Data Analytics Project 3: SQL Data Analysis

## Objective
The objective of this project is to analyze sales data using SQL queries. The project demonstrates SQL fundamentals including data selection, filtering, sorting, grouping, and aggregation to extract meaningful business insights from the dataset.

## Project Structure
- `Dataset for Data Analytics.xlsx`: The raw dataset containing 1,200 records of sales data.
- `Project_3_SQL_Data_Analysis.ipynb.ipynb`: The Jupyter Notebook containing the full end-to-end data analysis process, SQL queries, and visualizations.
- `sales.db`: An SQLite database generated during the analysis, containing the `orders` table.

## Dataset Information
The dataset contains 1,200 orders without any missing values. It includes 14 columns:
- **OrderID**, **Date**, **CustomerID**, **Product**, **Quantity**, **UnitPrice**, **ShippingAddress**, **PaymentMethod**, **OrderStatus**, **TrackingNumber**, **ItemsInCart**, **CouponCode**, **ReferralSource**, **TotalPrice**.

## Workflow and Data Preparation
1. **Load Dataset**: Read the Excel dataset into a Pandas DataFrame.
2. **Data Inspection**: Checked data types and validated there are no missing values.
3. **Database Creation**: Used SQLite3 to create a local database `sales.db`.
4. **Data Import**: Inserted the Pandas DataFrame into an SQL table named `orders`.
5. **SQL Querying & Visualization**: Executed 16 specific SQL queries and visualized the results using Matplotlib.

## SQL Queries Executed
This project successfully implemented the following SQL concepts: `SELECT`, `WHERE`, `ORDER BY`, `GROUP BY`, `COUNT()`, `SUM()`, `AVG()`, and `HAVING`.

Here is a detailed breakdown of the 16 queries performed in the analysis:
1. **Retrieve all records**: Fetched all rows from the `orders` table.
2. **Select specific columns**: Displayed only `Product`, `Quantity`, and `TotalPrice`.
3. **Filter high-value orders**: Used `WHERE TotalPrice > 1000` to find high-value orders.
4. **Filter by status**: Displayed all completed `Delivered` orders.
5. **Filter by payment method**: Displayed orders paid via `Credit Card`.
6. **Sort data**: Sorted orders by `TotalPrice` in descending order to see the highest value orders first.
7. **Count total orders**: Calculated the total number of customer orders.
8. **Calculate total revenue**: Used `SUM(TotalPrice)` to get the overall revenue.
9. **Calculate average order value**: Used `AVG(TotalPrice)` to determine the average customer spend.
10. **Count orders by product**: Grouped by product to find the number of orders for each product type.
11. **Calculate revenue by product**: Found revenue generated by each product, sorted from highest to lowest.
12. **Calculate average quantity sold per product**: Used `AVG(Quantity)` grouped by product.
13. **Count orders by payment method**: Grouped by payment method to understand customer payment preferences.
14. **Count orders by referral source**: Analyzed where customers came from (e.g., Instagram, Facebook).
15. **Calculate revenue by order status**: Analyzed revenue distribution across different statuses (Shipped, Returned, etc.).
16. **Filter grouped data**: Used the `HAVING` clause to display only products whose total revenue exceeds 20,000.

## Data Visualizations
To support data-driven decision making, the SQL results were visualized using Python's `matplotlib`:
- **Revenue by Product**: A bar chart displaying which products generate the highest revenue.
- **Orders by Payment Method**: A pie chart showing the percentage distribution of different payment methods.
- **Revenue by Order Status**: A bar chart comparing the revenue across different order statuses.
- **Orders by Referral Source**: A bar chart identifying the referral sources that bring the highest volume of customer orders.
